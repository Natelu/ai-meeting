# Kubernetes Middleware Helm Chart Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Provide a Helm chart for deploying AI Meeting middleware on Kubernetes with production-grade PostgreSQL failover, NodePort exposure, local disk persistence, and Whisper model loading from the existing MinIO S3 endpoint.

**Architecture:** PostgreSQL is represented as a CloudNativePG `Cluster` so failover and replication are handled by the operator. Redis, LiveKit, Whisper ASR, Prometheus, Grafana, and Loki are deployed directly by this chart with local hostPath-backed persistence where needed. Whisper model files are synchronized from the pre-existing MinIO/S3 endpoint by an init container before the ASR server starts.

**Tech Stack:** Helm v3, Kubernetes, CloudNativePG, PostgreSQL 16, Redis 7, LiveKit, Whisper ASR Webservice, MinIO S3 API, Prometheus, Grafana, Loki.
