# 实施路线图

## Phase 0: 需求收敛与选型

目标：

- 把产品边界和选型风险先压实

交付物：

- 确认目标客户画像与试点场景
- 确认 RTC、录制、ASR、LLM、存储方案候选
- 确认私有化部署方式
- 输出试点客户的部署前提清单

验收标准：

- 能回答“为什么客户会买”而不是只回答“我们能做什么”
- 每一个核心依赖都有备选方案

## Phase 1: 可部署会议基础盘

目标：

- 先让客户环境内可以稳定开会、录制、回放

范围：

- 会议创建与加入
- 多人音视频
- 屏幕共享
- 录制与回放
- 基础权限控制

验收标准：

- 20 到 50 人会议稳定运行
- 录制文件可回放
- 管理员可限制会议访问和回放访问

## Phase 2: AI 纪要与归档

目标：

- 建立会后处理主流程

范围：

- 录制完成事件处理
- ASR 转写
- 纪要摘要
- 待办提取
- 写入客户存储
- 检索索引构建

验收标准：

- 会后 5 分钟内可返回纪要和待办
- 失败任务可重试
- 用户可检索历史会议内容

## Phase 3: 企业接入与标准交付

目标：

- 让产品从“能跑”进入“能卖”

范围：

- 身份系统接入
- OA 或日程接入
- 存储配置向导
- 审计日志
- 部署文档和运维手册

验收标准：

- 新客户接入不依赖大规模定制开发
- 关键接入链路可通过配置完成

## Phase 4: 垂直场景增强

目标：

- 在不破坏核心定位的前提下增强差异化

候选方向：

- 结对编程会议模板
- 文稿评审会议模板
- 知识库联动
- 智能会前准备与会后追踪

进入条件：

- Phase 1 到 Phase 3 已证明客户愿意为主价值付费
- 交付复杂度受控

## 暂缓项

以下内容只有在主线稳定并商业成立后才考虑：

- 情绪识别
- 手势白板
- 音色模拟
- 通用协同编辑
